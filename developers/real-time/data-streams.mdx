## Available Streams

Polymarket provides several real-time data streams for different use cases.

## Market Data Stream

### Price Updates

Real-time price changes for markets:

```javascript
ws.subscribe({ channel: 'prices' });
```

Receive updates:
```json
{
  "type": "price_update",
  "market_id": "market_id",
  "token_id": "token_id",
  "price": "0.65",
  "change_24h": "+0.05",
  "timestamp": 1704067199
}
```

### Market Statistics

Live market statistics:

```json
{
  "type": "stats_update",
  "market_id": "market_id",
  "volume_24h": "5000",
  "trades_24h": 234,
  "liquidity": "10000",
  "timestamp": 1704067199
}
```

## Order Book Stream

### Full Order Book

Complete order book snapshots:

```javascript
ws.subscribe({
  channel: 'orderbook',
  token_id: 'token_id',
  depth: 10 // Number of price levels
});
```

Snapshot:
```json
{
  "type": "snapshot",
  "token_id": "token_id",
  "bids": [
    {"price": "0.65", "size": "100"},
    {"price": "0.64", "size": "200"}
  ],
  "asks": [
    {"price": "0.66", "size": "150"},
    {"price": "0.67", "size": "250"}
  ],
  "timestamp": 1704067199
}
```

### Incremental Updates

Delta updates for order book changes:

```json
{
  "type": "orderbook_update",
  "token_id": "token_id",
  "changes": [
    {
      "side": "bid",
      "price": "0.65",

  "timestamp": 1704067199
}
```

## Data Processing

### Stream Buffer

Handle bursts of messages:

```javascript
const buffer = [];
let processing = false;

ws.onmessage = (event) => {
  buffer.push(JSON.parse(event.data));

  if (!processing) {
    processing = true;
    processBuffer();
  }
};

function processBuffer() {
  while (buffer.length > 0) {
    const message = buffer.shift();
    handleMessage(message);
  }
  processing = false;
}
```

### State Management

Maintain local state:

```javascript
const state = {
  prices: {},
  orderBooks: {},
  myOrders: {}
};

function handleMessage(data) {
  switch(data.type) {
    case 'price_update':
      state.prices[data.market_id] = data.price;
      break;
    case 'orderbook_update':
      updateOrderBook(state.orderBooks, data);
      break;
    case 'order_update':
      state.myOrders[data.order_id] = data;
      break;
  }
}
```

### Throttling Updates

Limit UI update frequency:

```javascript
let lastUpdate = 0;
const updateInterval = 100; // ms

function handlePriceUpdate(data) {
  const now = Date.now();
  if (now - lastUpdate > updateInterval) {
    updateUI(data);
    lastUpdate = now;
  }
}
```

## Use Cases

### Real-Time Dashboard

Display live market data:

```javascript
ws.subscribe({ channel: 'prices' });
ws.subscribe({ channel: 'market_events' });

ws.on('price_update', (data) => {
  updatePriceDisplay(data.market_id, data.price);
});

ws.on('market_created', (data) => {
  addMarketToList(data);
});
```

### Trading Bot

Automated trading based on data:

```javascript
ws.subscribe({ channel: 'orderbook', token_id: 'xyz' });
ws.subscribe({ channel: 'user', auth_token: 'token' });

ws.on('orderbook_update', (data) => {
  const signal = analyzeOrderBook(data);
  if (signal) {
    placeOrder(signal);
  }
});

ws.on('order_update', (data) => {
  if (data.status === 'FILLED') {
    logTrade(data);
  }
});
```

### Portfolio Tracker

Monitor your positions:

```javascript
ws.subscribe({ channel: 'user', auth_token: 'token' });

ws.on('trade_notification', (data) => {
  updatePosition(data);
  calculatePnL();
});

ws.on('balance_update', (data) => {
  updateBalanceDisplay(data);
});
```

## Performance Considerations

### Message Rate

Typical message rates:
- Prices: 1-10 per second per market
- Order book: 10-50 per second per token
- Trades: 1-20 per second per market
- User updates: As events occur

### Bandwidth

Average bandwidth usage:
- Market stream: ~10 KB/s
- Order book stream: ~50 KB/s
- Trade stream: ~20 KB/s
- User stream: ~5 KB/s

### Latency

End-to-end latency:
- Price updates: 50-100ms
- Trade execution: 100-200ms
- Order updates: 100-300ms

## Best Practices

1. **Subscribe Selectively:** Only to needed streams
2. **Buffer Messages:** Handle bursts gracefully
3. **Throttle Updates:** Don't overwhelm UI
4. **Maintain State:** Track changes incrementally
5. **Error Handling:** Graceful degradation
6. **Reconnection:** Auto-reconnect on disconnect
